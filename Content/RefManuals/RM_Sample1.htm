Introduction

This guide describes how to install RegionModule Setup 1.2.3. It includes installation, uninstallation, key configuration steps, and references to the related release notes.

Important information

Review all prerequisites and configuration settings before starting the installation process. Ensure that the environment meets the required platform and licensing conditions.

Continuous release overview

As part of the transition to cloud-based enterprise solutions, this release introduces Generic ERP Continuous Update, replacing the legacy Generic ERP2. The updated version initially mirrors existing functionality while enabling a smooth transition to cloud infrastructure.

Updates for the legacy version will continue to ensure regulatory compliance.

Prerequisites

  • Product licence: The environment must have the correct licence for RegionModule Setup. This licence differs from the legacy version despite similarities in functionality.
  • To enable the Invoice listener feature:
    • In the Support portal, go to Platform services.
    • Enable the Identity service: Confirm the environment is set to Shared Cloud and the application is RegionModule Setup.
    • Enable the Extension service: Ensure the environment matches the Shared Cloud configuration.
    • Enable the Flow kit value for the Sample listener: Request activation via the Industry flow request section. Confirm that Extension Kit is enabled and select Invoice listener in the available flow options.

Tip: Detailed setup steps are available in the supplementary document InvoiceListenerSetup.docx.

Installation

This section explains how to install or upgrade RegionModule Setup and migrate users from an on-premises deployment to the cloud.

Note: Ensure the web server identity has read/write access to the folder defined by the LOG_PATH environment variable.

Warning: Confirm that the following entry exists in the configuration file:

			<sampleCode>
			...
			</sampleCode>

Running the installation script

  1. Extract the installation package.
  2. Open PowerShell as an administrator.
  3. Navigate to the extracted folder.
  4. Run Install.ps1 with the following parameters:
    • dataSource: Target database connection.
    • assemblyMode: Load method (for example, Normal, File, Database).
    • architecture: Target architecture (default: x64).
    • runScript: Execute mandatory scripts (required for fresh installations).
    • runTaxonomy: Install taxonomy codes.
    • runMarketplace: Install template marketplace.
    • migrate: Remove legacy system parameters.
    • client: Default client (for example, “Region”).
    • systemSetup: Default system setup value.
    • userId: Default user ID (for example, “SYSTEM”).
    • h: Display help information.
  5. For production environments, run InstallWeb.ps1 with:
    • milestone: Core version (default: 7).
    • iisreset: Restart IIS after installation (default: false).

Verifying installation

  • Check that the relevant screens appear in the application menu.
  • Confirm assembly versions in the application’s details section.
  • Verify that stylesheet files are copied to the folder defined by STYLESHEET_PATH.
  • Confirm that the handler process runs correctly and logs the correct export directory.

Licence setup

  • Install licence files in both the business server and web server directories.
  • Using punchout features without a valid licence triggers a warning.

Uninstallation

  • Run Uninstall.ps1 in PowerShell as an administrator.

Troubleshooting

Create local logging group

  1. Create a group named Logging.

  2. Add relevant service accounts (for example, ASPNET, IUSR, NETWORK SERVICE).

  3. Assign permissions to the folder defined by LOG_PATH.

Enable logging

  1. Open the Management Console.

  2. Navigate to logging settings.

  3. Select the preferred logging method (event log, database, or file).

  4. Set the appropriate priority level.

  5. Save the configuration.

Tip: Restart IIS or the business service after enabling logging.
