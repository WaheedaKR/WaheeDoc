Support note documentation – newsletter

This newsletter provides an overview of recent changes, improvements, and updates to the processes, tools, and documentation related to support note creation and maintenance. All updates are also summarised in the Summary of changes section within the Support notes overview.

New content updates

  • Versioning guidance added: A note has been added to clarify when to use major and minor versions in different scenarios.
  • Library selection warning: A warning now advises users to keep only one library open when using the drag-and-drop function to avoid attaching support notes to the wrong library.
  • Obsolete section removed: The section describing how to install a library from the information browser has been removed, as it is no longer relevant.
  • Content migration – “New functionality” section: The content previously located in the “New functionality” section has been moved to the section covering support note migration. This includes guidance on mass validity updates and reviewing support notes using tracked changes.
  • Approval button removed from editing tool: The “Set as approved” button has been removed from the summary section in the editing environment. As a result, the summary section has been updated and split into two subsections:
    • For product type A: The approval date is now automatically set to the most recent change.
    • For product type B: The approval date must be selected manually in the metadata section, regardless of the summary section dates.
  • Validity note added: A note has been added to clarify that the validity information in the impacted systems and software releases section includes currently active software. This section is reviewed before each new system release.

Updates to existing content

  • First-time user guidance: A note has been added for new users accessing the content management environment. It explains an additional step required in the search dialogue due to initial profile settings.
  • Updated screenshot for extract settings: The screenshot for setting extract conditions has been updated to clarify the difference between public and internal attributes.
  • Expanded database update steps: Additional steps have been added to the procedure for updating the content database, providing more detailed guidance.
  • Template creation note added: A note has been added to highlight the importance of using the “Create new from template” option when creating a new support note.
  • Obsoletion steps clarified: The procedure for marking a support note as obsolete now includes the missing step of updating attributes. Relevant screenshots have been added for clarity.
  • Title and keyword guidance: Instructions for writing titles have been updated. A note has been added to clarify when it is appropriate to remove comment tags from titles.
  • Impacted elements update: The procedure for adding impacted elements manually or using the helper tool has been added to the relevant step in the process.
  • Client key warning added: A note has been added to remind users not to change the client key when updating a support note directly in the content management environment.

Improvements

  • Extract options clarified: The extract options have been described in more detail. A warning has been added to prevent unintended changes to the version ID when using the edit function.
  • Content reorganisation: The subsection on updating the content database has been moved from the basic activities section to the support note creation section for better alignment.
  • Impacted elements procedure refined: Step-by-step instructions for completing the impacted elements section have been clarified and expanded.
  • Update procedure improved: An additional step has been added to the update process to ensure changes are saved correctly when editing a support note directly in the content management environment.
  • Paragraph section updated: A note has been added and the content has been improved for clarity and completeness.

Approval date handling

The “Set as approved” button has been removed for all support notes. For product type A, the approval date is automatically set to the most recent change. For product type B, the approval date must be selected manually in the metadata section, regardless of the summary section dates.

The approval date, which appears on the support note cover, must now be set manually in the metadata section. This change ensures consistency and accuracy in approval tracking.

Upcoming update window

The next update window is scheduled to begin at 07:00 (UTC) on 8 March 2021.

If you need to request updates to public or internal support notes, please contact the responsible content owners in advance. Include the following information in your request:

  • Support note ID
  • Version ID of the approved version

Once the update window begins, no further updates will be processed unless they are urgent or involve new support notes. Any regular updates submitted after the start of the window will be deferred to the next update window, scheduled for 22 March 2021 at 07:00 (UTC).

If you have any questions, please contact your local documentation coordinator.
