Internal retrieval (trx900)

This window is used to register internal retrieval of items from inventory. It is not possible to amend a retrieval after it has been saved. When saving, a retrieval number is allocated to an entry. This retrieval number can be used as a template for future item retrievals. The window displays the same item lines as in the original entry, and you can manually enter the number of units to be retrieved. When saving the transaction, a new request number will be assigned to it.

The inventory level will be updated automatically once the retrieval is saved. The basis for updating the financial ledger is saved in a specific register, and this is posted when the stockFlow processor has processed the transactions. The ledger analysis of the financial ledger transactions must be entered using the ledger mapping section or the Preset reference field. The default accounts are retrieved from the item category.

System configuration
  • Storage zones and locations must be defined in the storage configuration window.
  • Retrievable items must be entered in the item catalogue window.
  • The items to be retrieved must be in inventory.
  • One or more code types with the treatment code 91 (internal retrieval) must be set up in the code types window. Code types must be linked with valid code cycles (defined in the sequence manager window).
  • Persons authorised to carry out inventory retrievals must be set up in the access roles window under the shared settings module.
Basic usage instructions

Register an internal inventory retrieval

  1. If two or more code cycles are defined for code types with the treatment code 91, a code cycle must be selected in the cycle selector window which appears as a pop-up before saving the entry. The system automatically selects a code cycle if only one is available with the status active.
  2. Indicate the description and user with authorisation responsibility, and attach a reference if you want.
  3. Enter the transaction date.
  4. Enter the item code of the items that are to be retrieved from inventory in the item code field.
  5. To use a previously registered retrieval as a template, you can open the existing request by clicking the open button. The items that were registered will then be displayed in the table.
  6. When registering the first line, you will be asked to enter the ledger analysis. State how this line is to be posted. If a template has been defined, it is not necessary to set up a ledger analysis for the remaining lines.
  7. The combination of storage zone and location that is registered in the item window will be set up as the default. These can be overridden if you want to carry out a retrieval from another zone/location combination.
  8. If the item is split into batches or serial numbers, the batch/serial number of the items to be retrieved must be entered.
    Note: When you stock an item with the batch and date limit flag selected in the item catalogue window, and provide this item with a new batch (that doesn't already exist in inventory), a new pop-up window called batch details will appear. In this window, you can provide the date from and date to fields if required. This pop-up window will only appear for newly created batches. For an existing batch, you can use the batch details button to view its details.
  9. State the number of units to be retrieved. If the item is divided into serial numbers, the number must be equal to 1.
  10. Click the confirm button.
  11. On clicking save, the cycle selector pop-up window appears. Please select the desired code cycle to complete the registration process.
Tip: If you have selected a code series with automatic number allocation, the inventory transaction number will be automatically allocated. With manual assignment, you must allocate the inventory retrieval a transaction number.

Delete rows during registration

  • One or many items can be deleted from the request details.
  • Check the box towards the left of the target row which has to be deleted.
  • Click the remove button in the menu.
    Warning: You cannot delete a row that has been saved.
  • Click the confirm command in the data menu.
Window fields and options

Inside the Request information section

  • Status:Pending – The retrieval has not been posted.
  • Status:Finalised – The retrieval is posted. The status cannot be amended. On registration, the status is set to pending. It is amended to finalised by the stockFlow processor.
  • Description: Optional text posted in the financial ledger when the stockFlow processor is run.
  • Responsible: The user responsible for this transaction. Must belong to a role defined in access roles with responsibility type = general for the inventory control module.
  • Cycle: The accounting cycle in which the item was registered. Must be valid and defined in the cycle maintenance window. Defaults to the current cycle.
  • Date: The retrieval date. Defaults to the current date.

Inside the Request details section

  • Item code: Identifier for the item. Only listed inventory items can be used.
  • Description: Description of the item.
  • Preset reference: Used to retrieve a previously saved ledger analysis.
  • Storage zone: Zone from which the item is retrieved. Must be valid and defined in the storage configuration window.
  • Location: Specific location within the zone. May be mandatory or optional.
  • Batch ID: Identifier for batch-tracked items. Mandatory if tracked by batch.
  • Serial ID: Identifier for serial-tracked items. Mandatory if tracked by serial number.
  • Unit type: Unit of measure for the item. Defaults to base unit but can be overridden. Conversion factor is applied if a different unit is entered.
  • Unit value: Average inventory value of the item.
  • Note: This is indicative. If using rolling average or FIFO, this field shows average value. If using fixed cost, it shows standard cost. Actual value is determined by the valuation method and set by the stockFlow processor.
  • Quantity: Number of units to be retrieved. Must be 1 for serial-tracked items. Cannot exceed available inventory.
  • Total value: Calculated as unit value × quantity.

Inside the Ledger analysis section

This command sets up the ledger analysis for a line in the table. It requires the same input as the preset reference field but behaves differently in the following ways:

  • Applies only to the selected line.
  • Default attribute from the authorisation register is used (e.g., cost centre 230).

  • A preset may be retrieved and used as the foundation for line-level mapping. If defined, it will be reflected in the transaction line.

Note: If you modify the line, ensure the Preset reference field is cleared before saving. This action will override any existing preset mapping associated with the line.

Buttons in this window
  • Confirm: Save your changes.
  • Reset: Clear all unsaved fields.
  • Retrieve: Search and open a specific entry using the lookup pop-up window.
  • Duplicate: Copy your changes.
  • Inventory check: View inventory levels for the selected item.
  • Retrieval printout: Print the internal retrieval summary using the retrieval printout window.
  • Batch details: View batch information and associated dates for the selected batch.
